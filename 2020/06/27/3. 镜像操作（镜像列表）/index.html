3. 镜像操作（镜像列表）

win10安装Docker

  Docker官网，下载Docker Desktop并安装。

一、获取镜像

  默认从Docker Hub网站获取library用户下的ubuntu仓库的18.04版本镜像。

docker [选项] [Docker Registry 地址[:端口号]/]仓库名[:标签]

二、列出镜像

  1. 列出可用镜像

    docker image ls

  2. 列出玄虚镜像(相同名称的新镜像覆盖了就镜像，旧镜像成为了玄虚镜像)

    docker image ls -f dangling=true

  3. 删除玄虚镜像(一般无存在的意义了)

    docker image prune

  4. 列出所有镜像，包括中间层镜像（无标签，是一些镜像的基础镜像）

    docker image ls -a

  5. 列出部分镜像

列出ubuntu的所有版本

docker image ubuntu

指定列出该镜像的18.04版本

docker image ubuntu:18.04

  1. -f过滤器参数

列出在mongo:3.2镜像之后获取的镜像

docker image -f since=mongo:3.2

列出在mongo:3.2镜像之前获取的镜像

docker image -f before=mongo:3.2

此外，如果镜像构建时，定义了 LABEL，还可以通过 LABEL 来过滤。

docker image ls -f label=com.example.version=0.1

三、以特定格式显示

  1. 使用--format参数

    docker image ls –format “\t“

  2. -q（只显示镜像id）

    docker image ls -q

四、删除镜像

  删除镜像有两个模式，分别是Untagged和Deleted；因为镜像的唯一标志是其ID，当一个镜像有多个标签关联的时候，根据标签删除镜像，如果除了被指定删除的标签之外还有其他标签指向该镜像，则该镜像不会被删除；如果有容器依赖于该指定删除的镜像，则该镜像也不会被删除；如果有镜像以此镜像作为基础镜像，该镜像也不会被删除。

docker rm [选项] <镜像1>[<镜像2> …]

使用docker image ls配合删除镜像
比如，我们需要删除所有仓库名为 redis 的镜像：

docker image rm $(docker image ls -q redis)

或者删除所有在 mongo:3.2 之前的镜像：

docker image rm $(docker image ls -q -f before=mongo:3.2)
