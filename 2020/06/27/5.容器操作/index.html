5.操作容器

容器

  镜像的实例，独立运行的一个或一组应用，以及他们运行时的环境。虚拟机则可以理解为模拟运行的操作系统和运行在其上的程序。

一、查看容器状态

查看正在运行的容器

docker container ls

查看所有的容器

docker container ls -a

二、启动容器

新建启动

docker run ubuntu:18.04 echo ‘Hello World’

新建一个终端交互容器

docker run -it ubuntu:18.04 bash

  当执行docker run时，Docker在后台首先会检查是否有对应镜像存在，如果没有会下载。利用镜像创建一个容器、分配文件系统，在此基础上挂载一层可读写层、从宿主的主机配置网桥接口中桥接一个虚拟接口到容器、从地址池分一个ip池给容器、执行应用程序、执行完终止。

启动已终止容器
启动一个已终止的容器。

docker container start

三、删除容器

删除指定容器

docker container rm [container ID or NAMES]

使用-f参数，删除正在运行的容器。Docker会发送SIGKILL信号给容器。

清空容器

docker container prune

四、进入容器，docker attach和 docker exec

使用了-d参数的容器以守护态运行，当需要进入容器操作时，可以使用docker attach或者docker exec命令。

使用docekr attach时，执行exit时会结束容器的运行。

docker attach [container ID or NAMES]

使用docker exec -it [container ID or NAMES] bash时，执行exit不会导致容器结束运行。

docker exec -it [container ID or NAMES]

建议使用docker exec -it [container ID or NAME] bash
