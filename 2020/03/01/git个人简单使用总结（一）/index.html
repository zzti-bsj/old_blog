git简介

  git是一个分布式版本控制器。
  git是面向修改的。所谓的面向修改，比如修改文件的一句代码、修改了文件名、新建一个文件，这些都是修改。

工作区和版本库、暂存区

  这里要简单说一下工作区、版本库和暂存区三个区分区别。

  • 工作区：文件夹里能够看到的，文件夹仓库就是工作区
  • 暂存区(stage)：在版本库中的一个空间，称为暂存区，记录着git add的文件修改
  • 版本库(Responsitory)：.git是git的版本库。

初始化本地git仓库

  • git init

添加、提交文件

  添加和提交文件需要经过两个步骤。
  1. git add <filename>添加文件修改到暂存区中。
  2. git commit -m "提示信息"提交暂存区中的文件到版本库中。

撤销修改

  1. 撤销暂存区的修改
    git reset HEAD filename

  2. 撤销工作区的修改
    git checkout -- <filename>

回滚：回滚的是版本库中的commit的各个版本。
撤销：把在工作区的修改全部撤销。
1. 修改后还没有放入暂存区，撤销修改之后就和当前版本库中的一模一样。
2. 修改后放入暂存区后，又作了修改，这时候撤销修改就回到添加到暂存区后的状态。

总之，就是让这个文件回到最近一次git add或git commit的状态。

撤销修改的例子

回滚和撤销的例子

  • 情况1：自修改后还没有被放到暂存区，现在，撤销修改就回到和版本库一模一样的状态；
  • 情况2：已经添加到暂存区后，又作了修改，现在，撤销修改就回到添加到暂存区后的状态

情况1：

  • git checkout -- <filename>

情况2：

  • git reset HEAD <file>，把暂存区的修改撤销到与当前版本库相同（一键还原），重新放回工作区；
  • git checkout -- file，丢弃工作区的修改。

版本回退

回滚到某个命令
git reset --hard commmitid 回滚到某一个版本

回滚命令

  • git reset --hard^ 回滚到上一个版本
  • git reset --hard^^ 回滚到上上一个版本
  • git reset --hard~100 回滚到上一百个版本

删除文件

删除工作区的文件

  • rm <filename>

删除版本库中文件

  • git rm <filename>
  • git commmit -m "提交信息"

没有被添加到版本库中就被删除的工作区的文件，是无法恢复的
