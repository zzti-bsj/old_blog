协程(单进程的异步编程模型、单线程)

  协程：Coroutine
  子程序调用：递归时经常会调用自己，中断而去执行更深层的函数，结束最底层的函数执行之后层层返回往下接着执行！
  协程：协程不同于子程序调用，协程没有经过函数调用。比如在执行A()一段时间后，突然切换去执行B()，这样看着像是多线程，分别执行A()、B()，但实际上只使用一个线程执行。

协程优势

  以生产者消费者问题为例，一个线程负责生产，另一个线程负责消费，这个时候涉及到线程锁，以及线程切换，极大的浪费了时间；而使用协程，因为是单线程，所以不需要线程切换，省去了切换线程花费的时间，并且不涉及到数据互斥锁问题；
  在一个问题中，线程的数量越多，使用协程的优势就越明显。
